[[image-vision-example]]
= Image Vision Example

This Spring AI example contains a simple Spring Boot application using Spring AI's https://docs.spring.io/spring-ai/reference/api/chatclient.html[Chat Client API] ask the AI model to find Waldo in an image.

In this example, I tested with several different AI models and ending with OpenAI's `gpt-4o-mini` model, Ollama's `llama3.2-vision` model from Meta and Mistral's `pixtral-12b-2409` model.

I tested with 3 different Where's Waldo images depicting a https://github.com/PacktPublishing/Building-Intelligent-Applications-With-Spring-AI/blob/main/image-vision-example/src/main/resources/WheresWaldoCityScene.png[city scene], a https://github.com/PacktPublishing/Building-Intelligent-Applications-With-Spring-AI/blob/main/image-vision-example/src/main/resources/WheresWaldoGardenScene.png[garden scene] and a https://github.com/PacktPublishing/Building-Intelligent-Applications-With-Spring-AI/blob/main/image-vision-example/src/main/resources/WheresWaldoGardenScene.png[winter scene].

I used the prompt, "_Evaluate the given image and find the exact location of Waldo Describe the precise location in detail_"

When ran, the results were less than impressive. Each AI model did a nice job of explaining the scene in the image, but none of the models could find Waldo. Many stated as such, but in some cases, the AI model stated the wrong location. In 1 case, I used a different image that contained a person dressed up partly like Batman. I asked AI to not only find the person who dressed up like Batman, but generate a new image from the given image and circle the person dressed like Batman in red. Instead of finding and circling the correct person, it generated a figure of Batman in a completely different location and circled the character in red. In other words, it cheated! I guess humans and AI are not all that dissimilar after all.
